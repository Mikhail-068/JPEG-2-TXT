# JPG/PDF -> TXT

Веб-приложение для распознавания текста из PDF/JPEG/PNG/WEBP через уже развернутую Qwen Vision в контейнере `nm-ollama`.

## Что делает

- Загружает PDF или изображение.
- PDF рендерит постранично в JPEG.
- Отправляет изображение/страницы в Ollama API `http://nm-ollama:11434/api/chat`.
- Использует модель `qwen3.6:35b-a3b-q4_K_M`.
- Возвращает распознанный текст в поле TXT.
- Позволяет менять промпт через модальное окно. Промпт хранится в `data/prompt.txt`, поэтому сохраняется между перезапусками контейнера.

Стартовый промпт:

```text
Перепиши весь текст.
```

## Запуск на Monitosha

Контейнер нужно подключить к той же сети, где уже работает `nm-ollama`: `nm-monitor_default`.

```bash
docker compose up -d --build
```

После запуска сайт будет доступен на:

```text
http://<server-ip>:8090
```

Проверка:

```bash
curl http://127.0.0.1:8090/health
```

## Настройки

Основные переменные окружения в `docker-compose.yml`:

- `OLLAMA_BASE_URL` - адрес Ollama внутри Docker-сети.
- `OLLAMA_MODEL` - имя модели Qwen.
- `PDF_DPI` - качество рендера PDF-страниц.
- `MAX_PDF_PAGES` - ограничение страниц PDF за один запрос.
- `OLLAMA_NUM_CTX` - контекст модели для OCR-запроса. По умолчанию `4096`, чтобы на RTX 4090 оставался запас VRAM под обработку изображения.
- `OLLAMA_NUM_PREDICT` - максимальная длина ответа модели в токенах.
- `MAX_IMAGE_SIDE` - максимальная сторона изображения перед отправкой в Qwen.
- `JPEG_QUALITY` - качество JPEG после нормализации изображения.
- `OLLAMA_TIMEOUT_SECONDS` - таймаут ожидания ответа модели.
